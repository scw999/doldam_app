# 디자인 워크플로우 — Claude 웹 ↔ RN 구현

## 3단계 루프

```
① [Claude 웹]          ② [사용자]              ③ [Claude 터미널(나)]
─────────────────      ──────────────          ─────────────────
design-brief.md   →    받은 React 코드   →     RN 포팅 + theme
+ "이번 요청"          복붙해서 전달             업데이트
복붙 → 디자인 요청
```

## 단계별 상세

### ① Claude 웹 Artifact에서 디자인 받기

1. **claude.ai** 열고 새 대화 시작
2. `docs/design-brief.md` 내용을 **통째로 붙여넣기**
3. 맨 밑 "이번 요청" 섹션만 교체:
   ```
   화면 이름: 홈 스크린
   필요한 요소:
   - 상단 브랜드바 (돌담 로고, 187P 칩, 알림)
   - 오늘의 기분 이모지 6개
   - 포인트 카드 (187P + "방 개설까지 13P" 프로그레스 바)
   - 🔥 이번 주 핫 투표 랭킹 3개
   - 💬 최근 인기글 2개
   - 하단 5탭

   특히 신경 써줄 것:
   - 포인트 카드는 그라데이션 primaryDark → primary
   - 기분 이모지 선택 시 scale 1.15 + accent 배경
   ```
4. Claude가 React 컴포넌트를 Artifact로 그려줌 → **미리보기로 시각 확인**
5. 피드백 주면서 반복 ("그림자 줄여", "폰트 더 크게", "투표 카드 순위를 원형 뱃지로")

### ② 나(터미널 Claude)에게 전달

확정된 Artifact의 **React 코드 전체를 그대로 복사**해서 이 대화창에 붙여넣고:

```
이거 RN으로 포팅해줘. 파일: apps/mobile/app/(tabs)/index.tsx
```

### ③ 내가 하는 일 (자동)

- `COLORS` 객체 → `apps/mobile/src/theme.ts` 병합
- `<div>/<p>/<button>/<input>` → `View/Text/Pressable/TextInput` 변환
- inline style → `StyleSheet.create`
- `background: linear-gradient(...)` → `expo-linear-gradient`
- 목업 const → 실제 API 호출로 교체 (`api.get(...)`)
- 네비게이션 placeholder → `router.push(...)`
- 결과를 실제 파일에 Write

## 화면별 요청 예시 템플릿

### 홈
```
화면: 홈 (apps/mobile/app/(tabs)/index.tsx)
요소: 상단 브랜드바 / 오늘의 기분(6이모지) / 포인트 카드+프로그레스 /
     🔥 이번 주 핫 투표 랭킹 Top3 / 💬 최근 인기글 2개
특히: 포인트 카드 그라데이션, 기분 선택 시 +3P 토스트
```

### 게시판
```
화면: 게시판 (apps/mobile/app/(tabs)/board.tsx)
요소: 제목 + 카테고리 가로 스크롤 칩 6개 / 글 카드 리스트 /
     하단 "이 주제로 대화하고 싶다면?" 매칭 배너
특히: 글 카드에 공감 반응 이모지 4종(💛🫂💪😂) 카운트 표시
```

### 투표 상세
```
화면: 투표 상세 (apps/mobile/app/vote/[id].tsx)
요소: 질문 / 찬반 큰 버튼 2개 (⭕/❌) / 투표 후 결과 바(성별 분포) /
     공유카드 만들기 CTA / 500명+이면 🔥 테마방 배너
특히: 투표 후 결과 애니메이션 transition 800ms
```

### 채팅방
```
화면: 채팅방 상세 (apps/mobile/app/room/[id].tsx)
요소: 상단 방 정보(멤버/남은시간) + 유지/폭파 투표 버튼 /
     방 요약 카드(무료) / 멤버 리스트 + 🔒 정보열람 / 메시지 버블 / 입력
특히: 내 bubble은 primary 배경, 상대는 tag 배경. 남은시간 24시간 미만이면 빨강
```

### 마이
```
화면: 마이 (apps/mobile/app/(tabs)/my.tsx)
요소: 프로필 카드(아바타🌿 + 별명 + ✓인증 뱃지 + "이혼 2년차") /
     포인트 187P / 매칭 온도 36.5° / 뱃지 3개 + 🔒 더보기 /
     메뉴 리스트(내 글, 스크랩, 감정 타임라인, 체크리스트, 설정)
특히: 매칭 온도는 tinder 스타일 동적 지표 — 활동량 기반
```

## 팁

- **한 번에 한 화면만**. 탭 5개 한 번에 요청하면 품질 떨어짐
- Claude 웹에서 미리보기가 마음에 안 들면 "이 부분 더 여백 크게", "이 색 말고 accent로" 같이 부분 수정 요청 — 재생성 말고
- 받은 코드에서 **목업 데이터 배열은 건드리지 말고** 복붙해 줘. 내가 API 연결로 바꿈
- 사용자가 그림 그림을 첨부하고 "이 방향으로 해줘" 해도 OK — 이미지 기반 디자인 가능

## 이미 확보한 프로토타입

첨부해주셨던 프로토타입(단일 파일 전체 앱)은 전체 구조 참고용으로 `docs/design-reference.tsx`로 저장해두면 좋음. 화면별 분리 작업 시 베이스라인으로 사용.
