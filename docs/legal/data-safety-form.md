# Google Play 데이터 안전성 양식 — 답변지

Google Play Console → 앱 콘텐츠 → 데이터 안전 섹션 입력용. 각 카테고리별로 정확한 답변을 미리 정리해서 컨솔에서 그대로 옮기면 됩니다.

---

## 1. 데이터 수집 및 공유 — 기본 답변

| 항목 | 답변 |
|---|---|
| 앱이 사용자 데이터를 수집하거나 공유합니까? | **예** |
| 모든 사용자 데이터가 전송 중에 암호화됩니까? | **예** (HTTPS/TLS) |
| 사용자에게 데이터 삭제 요청 방법이 제공됩니까? | **예** (앱 내 탈퇴 메뉴) |
| 보안 검토에 따라 Play에서 데이터를 검증했습니까? | 신청 가능 (보안 모범 사례 준수 시 배지 부여) |

## 4. 데이터 사용·처리 — 세부 답변

각 카테고리에서 "수집"으로 표시한 항목별로 다음 답변 입력:

### 사용자 ID
- **수집됨**: 예
- **공유됨**: 아니오
- **암호화됨 (전송 중)**: 예
- **사용자가 데이터 삭제를 요청할 수 있나요?**: 예
- **수집 목적**: 계정 관리, 앱 기능
- **필수/선택**: 필수

### 전화번호
- **수집됨**: 예
- **공유됨**: 아니오 (SMS 발송 시 사업자로 전달 후 즉시 삭제 — Google 기준상 "공유"가 아닌 처리 위탁)
- **암호화됨**: 예
- **삭제 가능**: 예
- **목적**: 계정 관리(본인 확인)
- **필수**: 필수
- **보충 설명**: 원본은 저장하지 않으며 SHA-256 해시값으로만 보관

### 사진 (혼인관계증명서)
- **수집됨**: 예
- **공유됨**: 아니오
- **암호화됨**: 예
- **삭제 가능**: 예 (검토 완료 후 자동 삭제, 미완료 시 7일)
- **목적**: 계정 관리(자격 확인)
- **필수**: 필수

### 앱 상호작용 / 사용자 생성 콘텐츠
- **수집됨**: 예
- **공유됨**: 아니오
- **암호화됨**: 예
- **삭제 가능**: 예 (탈퇴 시 익명화 보존, 개별 글 삭제 가능)
- **목적**: 앱 기능, 분석
- **필수**: 필수

### 구매 내역
- **수집됨**: 예
- **공유됨**: 아니오
- **암호화됨**: 예
- **삭제 가능**: 회계법령상 5년 보관 의무
- **목적**: 앱 기능 (포인트 충전 내역)
- **필수**: 선택 (결제 시에만)

### 충돌 로그·진단 정보
- **수집됨**: 예
- **공유됨**: 아니오
- **암호화됨**: 예
- **삭제 가능**: 익명 처리되어 별도 삭제 불필요
- **목적**: 분석 (앱 안정성 개선)
- **필수**: 선택

### 기기 ID (푸시 토큰)
- **수집됨**: 예
- **공유됨**: 아니오
- **암호화됨**: 예
- **삭제 가능**: 예 (앱 알림 거부 또는 탈퇴 시)
- **목적**: 앱 기능 (푸시 알림)
- **필수**: 선택

## 5. Google Play 컨솔 입력 시 주의사항

1. **"공유"의 정의**: 위탁 처리(SMS 사업자, Cloudflare 등)는 Google 기준상 "공유"가 아닌 "수집된 데이터의 처리 위탁"으로 분류 — "공유" 답변은 "아니오"로
2. **변경 시 갱신**: SMS·IAP·AI 모더레이션 활성화 시 데이터 안전성 양식 업데이트 필수
3. **개인정보 처리방침 URL**: 반드시 외부 호스팅된 정적 URL 필요 (Notion 공개 페이지·GitHub Pages·자체 사이트 등)
4. **앱스토어와 일관성**: Apple App Store의 영양성분 라벨(Privacy Nutrition Label)도 동일한 정보로 작성해야 함

## 6. Apple App Store Privacy Nutrition Label 매핑

Apple은 카테고리 분류가 다름. 다음으로 매핑:

| Apple 카테고리 | 사용 여부 | Google Play 대응 |
|---|---|---|
| Contact Info | ✅ (Phone Number) | 전화번호 (해시 처리) |
| Health & Fitness | ❌ | — |
| Financial Info | ✅ (Purchase History) | 구매 내역 |
| Location | ❌ | — (시·도 수준 = "Coarse Location" 아닌 일반 텍스트 입력) |
| Sensitive Info | ❌ | — |
| Contacts | ❌ | — |
| User Content | ✅ (Photos, Other User Content) | 사진(증명서), 사용자 생성 콘텐츠 |
| Browsing History | ❌ | — |
| Search History | ❌ | — |
| Identifiers | ✅ (User ID, Device ID) | 사용자 ID, 푸시 토큰 |
| Purchases | ✅ | 구매 내역 |
| Usage Data | ✅ (Product Interaction, Crash Data) | 앱 상호작용, 충돌 로그 |
| Diagnostics | ✅ (Crash Data, Performance Data) | 진단 정보 |
| Other Data | ❌ | — |

각 항목에서 "Data Used to Track You"(추적용)는 모두 ❌, "Data Linked to You"(개인 연결됨)는 위 ✅ 항목들, "Data Not Linked to You"(익명)는 충돌 로그·진단 정보로 분류.
